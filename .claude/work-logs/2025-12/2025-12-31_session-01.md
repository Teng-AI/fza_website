## Key Decisions Made

### Decision 1: Download Images vs. Hotlink from Original Site

**Context**: The page needed images for MCs, team members, chapters, and venue. Options were to download and host locally or link to the original site.

**Options considered:**
1. Hotlink to original fuzhouamerica.org URLs - Simple, but creates external dependency
2. Download and host locally - More work, but ensures long-term reliability

**Decision**: Download and host locally

**Rationale**:
- Ensures images remain available even if the original site changes
- Improves page load performance (served from same domain)
- Provides full control over image optimization and formatting
- Eliminates external dependencies that could break

### Decision 2: Directory Structure for Conference Images

**Context**: Needed to organize 15+ images in a logical, maintainable way.

**Options considered:**
1. Flat structure - All images in `/public/images/conference/2023/`
2. Categorized subdirectories - Separate folders for mcs, team, chapters, etc.

**Decision**: Categorized subdirectories

**Rationale**:
- Better organization for maintenance
- Easier to locate specific image types
- Scalable for future conferences
- Mirrors the semantic structure of the page sections

### Decision 3: Sponsors Section Display Format

**Context**: Sponsor logos were not easily extractable from the original site.

**Options considered:**
1. Skip sponsors section entirely - Fastest but disrespectful to sponsors
2. Use placeholder images - Looks incomplete
3. Text-based display with sponsor names - Simple, clean, functional

**Decision**: Text-based display with styled sponsor names

**Rationale**:
- Ensures all sponsors are properly credited
- Maintains professional appearance
- Avoids copyright concerns with logo usage
- Still provides full sponsor recognition
- Can be upgraded to logos later if images become available

## Problems Solved

### Problem 1: Broken YouTube Video Embeds

**Issue**: Both video embeds on the page were not working (showing incorrect video or error). The 2022 Conference Recap and NYC Picnic videos were essential content but broken.

**Solution**:
- Inspected the original site's YouTube embed URLs
- Extracted correct video IDs from the original page
- Updated both iframe src URLs with correct IDs:
  - 2022 Recap: `eN_dGTTWizE`
  - NYC Picnic: `VjbWsyUwIo8`

**Learning**: Always verify video IDs by checking the original source. YouTube IDs are case-sensitive and must be exact.

### Problem 2: Venue Image Wrong Extension

**Issue**: Code referenced `venue.jpg` but actual downloaded file was `venue.png`, causing broken image display.

**Solution**: Updated the image path in the component from `.jpg` to `.png` to match the actual file extension.

**Learning**: When downloading assets, verify file extensions immediately and update code references accordingly to prevent broken links.

### Problem 3: Image Asset Organization

**Issue**: 15+ images needed to be downloaded and organized without clear initial structure.

**Solution**:
- Created logical subdirectories based on page sections (mcs, team, chapters)
- Downloaded images systematically by category
- Verified each image path in the code matched the file location

**Learning**: Organizing assets into categorical folders from the start saves time and makes the codebase more maintainable.
